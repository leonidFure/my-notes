# Нормализация
Нормализация - это процесс организации данных для уменьшения избыточности и предотвращения аномалий обновления.
Нормальные формы:
- 1NF: Все значения атомарны (нет списков или повторяющихся групп).
- 2NF: Удовлетворяет 1NF, и каждый неключевой атрибут зависит от всего первичного ключа.
- 3NF: Удовлетворяет 2NF, и нет транзитивных зависимостей.
# Денормализация:
В некоторых случаях данные специально денормализуются для увеличения производительности (например, чтобы уменьшить количество `JOIN` запросов).
